Introduction: Goals and Purpose

In my final project proposal I have proposed Personal Portfolio Website as my final project where the goal was to showcase my sister's art and photography creativity in a three-page navigation. The website will include most of her projects and skills as well as her biography and interests.

I have accomplished this objective by mainly designing and developing a portfolio website to interface with her viewers. I have applied as much what I’ve learned from the Fundamentals of Website Development (CSCI E-12) course as possible where HTML5, CSS, JavaScript, jquery, Ajax, multimedia functionalities, and co-ordinate content with those functionalities to produce a robust and user-friendly website.

Target Audience

As defined in my previous proposal, and after implementing the real website knowing that I have successfully accomplished my purpose and goal having a user friendly, robust, rich with gallery and entertaining website, I can verify here that Dania’s audience are:

  • Primary
    1. Architects and Designers, who have similar interests (whether it's a hobby or work) as Dania's, of both genders, all ages.
    2. High profile customers, of all genders and ages, looking for photos taken in a specific event, buy some of Dania's unique art pieces, etc.
    3. Potential Employers, where some art businesses might want to have Dania join their team.
    4. Art collectors, who are in love with art, can be in all genders and ages, retired people or youth.
    5. Future Competitors
  • Secondary
    1. Fans, Friends & Family

Item #1: Home: welcome audience, provide updated news, etc.

Accomplished

URL: http://cscie12students.dce.harvard.edu/~fkhalil/CSCIE12/Final_Project/FinalProject/home.php My Home Page is simple but very informative and eye catcher. Why?
  1. Navigation: Very simple navigation and clear font at the bottom of the page with a submenu for the gallery that expands upwards
  2. Easy Access: It’s easy to navigate throughout the pages via a fixed navigation bar that contains the navigation as well as a main title as a hyperlinks that takes the viewer to home page by clicking on it. This way I have accomplished a user friendly and easy to navigate website from one page/section to another. Hence, I can describe my navigation as bidirectional.
  3. Audio player: Music loads automatically when the page loads, with an audio control pad that is universal and easy to use. This audio tool bar is fixed on all pages. Knowing that there are songs gallery, I have implemented a jquery script that would trigger a “pause” when a user clicks on the song to play from the gallery, and when the user clicks on a regular picture another trigger would fire to play again.
  4. News update: Updated news in a carousal form with indicators to access it.
  5. Theme and Motto: Animation of the boat and the bar at the top of the screen with the music gives the viewer the sense of motion and entertainment.
  6. I have implemented my website using bootstrap theme and components, with iterations of the design and functionality.
  7. Personal Information & skills as “Biography”: Her “About” page is clear and concise to avoid any confusion about the function of her website. The well known “about me” is included in the home page in an unconventional way, where a modal pops up by clicking on a “Discover Me” button. This makes the viewer more curious into learning more about Dania.

Lessons Learned

I’m very excited as I’ve always wanted to be able to design and create websites. Taking this course and going through all the assignments we’ve been working was very helpful. Now, with this project I have learned a lot. Everything works well to an extent, and I have realized that there is always window for improvement as long as we have the backbone and structure of what we need to accomplish. Several functionalities that I have implemented can be done in different ways and many more that still needs to be added. In this section I will be sharing with you my thoughts.

The items that I’m pleased with are:

  1. The home page conveys the purpose and vision of the website clearly, and interacts well with the user.
  2. The flexibility in turning the automatic background audio off and on is very handy.
  3. Easy to maintain due to PHP structure that I built, where the admin can easily add/remove media files, and change fancybox presentation.
  4. Easy navigation through slides in the Gallery.
  5. Easy to figure out where the viewer is within the website through the breadcrumb added to each page.
  6. Reaching out to Dania is very friendly and easy.

The items that I’m pleased with are:

  1. The display of the gallery (photos, artwork and songs) can be done differently, making it faster to load.
  2. I used CSS only for the grow feature (Zooming) of images when hover over media items. Maybe with further tweaking with CSS, a border can be defined so as not to have a visible overlapping of images for the last item in every gallery page. Also if javascript was used it might be smoother.
  3. The contact details (locate me, connect with me on Linkedin and Facebook) can be illustrated in a more attractive way.
  4. Controlling the audio player was done by using javascript. Currently if a user clicks on regular images the music keeps playing. But when the user clicks on a song, the player will pause which is the desired outcome. However, when the user closes the youtube/song layout, the player is expected to play again, but now it doesn’t as I couldn’t find the javascript Event Method referring to “close”.

If I was given the chance to approach this project again, I might end up doing the same design structure. Might go different with theme colors though. What helped me in this enjoying experience is adopting the indentation and commenting practice since the very beginning when I started writing the code. So I will definitely stick to this approach as I recognized how important it is as I progressed and things got more complicated.

Future Directions

I kind of accomplished my vision for this project. As I have mentioned in my proposal, continuous improvement is necessary to make Dania’s website more effective in serving it’s function. Several features can be added in the near future which would enhance the interaction between Dania and her viewers. Some of these features are:
